Golang 踩坑和原理

发表于 2019-06-17 | 分类于 Golang

在实习的时候学习了 Golang，发现这是一门用起来非常爽的语言，记录一下学习过程。

函数调用原理

函数调用往往具有这种形式

C 语言函数调用原理

C 语言调用一个函数，函数的参数是通过寄存器和栈传递的，函数的返回值是通过 eax 寄存器传递的。所以 C 不支持多值返回。

Golang 函数调用原理

GO 语言传递和接受参数使用的是栈，这样的实现和使用寄存器相比性能会有一些损失，但是可以带来很多好处:

  1. 不需要考虑参数超过寄存器个数应该如何传递
  2. 更兼容不同硬件
  3. 函数可以多值返回

还有一个要注意的点是 Golang 的设计是: 无论传递基本类型，结构还是指针，都会对传递的参数进行拷贝。也就是说，不存在传值还是传引用的问题，一律拷贝。

总结

  1. 通过堆栈传递参数，入栈顺序从右到左
  2. 函数返回值通过堆栈传递，并预先分配内存空间
  3. 调用时都是传值，接收方会对入参进行复制再计算

  1. 如果期望容量大于当前容量的两倍就会使用期望容量；

Map 的实现原理

哈希表简介

哈希表能根据键(Key)直接访问内存中的存储位置，哈希表的实现关键在于选取哈希函数。在理想情况下，哈希函数应该能将不同的键映射到唯一索引上。这个理想状况是不太可能实现的。

解决冲突:

  1. 开放寻址法: 在一维数组对元素进行探测和比较以判断待查找的目标键是否存在当前哈希表中，如果发生冲突，将键值对写入下一个不为空的位置

  2. 拉链法:使用数组加链表实现哈希表，发生冲突时在链表末尾追加一个新的键值对

访问原理

在类型检查和中间代码生成阶段

当接受一个参数时，调用的时 mapaccess1， 两个时，调用的是 mapaccess2

每个桶都存储了 8 个 tophash，存储的是 hash 最高 8 位，每次都会与桶中全部 8 个 tophash 比较。选择桶时存储的是桶掩码的最低几位。这样能减少碰撞。

每个桶都是一整片的空间，当发现某一个 topbits 与传入的 tophash 匹配时，通过指针偏移量获取存储中的键进行比较，如果相同就返回。

使用 v, ok := hash[k] 的形式访问哈希表中元素时，我们能够通过这个布尔值更准确地知道当 v == nil 时，该空指针到底是哈希中存储的元素还是表示该键对应的元素不存在，所以在访问哈希时，更推荐使用这一种方式先判断元素是否存在。

写入

当 hash[k] 出现在左侧时，就会在类型检查和中间代码生成时被转换成调用 mapassign 函数。

首先根据传入的键拿到对应的桶

然后遍历比较桶中的 topshash 的键，如果找到了相同结果的就会获取目标的地址并返回。

当前哈希表没有处于扩容状态且装载银子已经超过 6.5 或存在太多溢出的桶时，就会调用 hashGrow 进行扩容。

如果当前的桶已经满了，就会调用 newoverflow 创建一个新的桶或使用 hmao 在预先 noverflow 中创建好的桶来保存数据。

扩容

扩容分两种，如果是溢出桶太多造成的，就会调用 sameSizeGrow

上面函数只是创建了新桶，还没有进行数据拷贝，数据拷贝是在 evacuate 函数中进行的

evacuate 函数在最开始创建一个用于保存分配目的 evacDst 结构体数组

如果这是一次不改变大小的阔日你哥， evacDst 结构体只会初始化一个，当哈希表容量翻倍时，一个桶中的元素会被分到新创建的两个桶中

删除

使用 delete 关键字删除

总结

GO 语言中使用拉链法来解决哈希碰撞的问题实现了哈希表，访问、写入和删除等操作都在编译期间被转换成了对应的运行时函数或者方法。

哈希在每一个桶中存储键对应哈希的前 8 位，当对哈希进行操作时，这些 tophash 就成为了一级缓存帮助哈希快速遍历桶中元素，每一个桶都只能存储 8 个键值对，一旦当前哈希的某个桶超出 8 个，新的键值对就会被存储到哈希的溢出桶中。

随着键值对数量的增加，溢出桶的数量和哈希的装载因子也会逐渐升高，超过一定范围时就会触发扩容操作，扩容时会将桶的数量分配，元素再分配的过程也是在调用写操作时增量进行的，不会造成性能的瞬时巨大波动。
