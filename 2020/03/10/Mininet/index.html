1.3 测试主机的连通性

观察 OPenFlow 信息流，h1 的 ARP 包查询 h2 的 MAC 地址，并同时方 packet_in 到控制器中，然后控制器就会广播 packet_out 信息到网络中， h2 主机收到这个 ARP 请求后，向对方回复，这个回复也会被发送到控制器中，然后控制器就会把这个数据包转发给 h1，并且下发一个流表到相关交换机中。

重复上一次 ping 操作发现第二次 ping 操作延迟降低很多，很明显，因为一条包含 ICMP ping 的流表已经下发到交换机中了，所以不需要产生新的控制流，并且这一类数据可以直接通过交换机。

2.2 设置网络连接选项

如果每段链路的延迟是 10 ms，由于 ICMP 请求走过了两端链路，返回也是两端，这个往返时延就是 40 ms。可以用 Mininet 的Python API 定义每一段链路

2.3 自定义网络拓扑

使用 python API，自定义网络拓扑是很容易的

当一个自定义的拓扑文件被提供时，能在命令中添加新的拓扑，新的交换机种类，以及测试刚刚创建的网路

2.4 自定义 MAC 地址

默认地，主机在启动的时候会被随机指派 MAC 地址。这样会使 debug 变得更加困难，因为每一次启动 Mininet 时，MAC 地址都会被改变，故某些主机之间的控制流会变得难以传输。

–mac 选项是十分有用的，并且能把主机的 MAC 地址和 IP 地址设置成为简单的、唯一的、便于阅读的 ID。

3.2 Mininet.link 模块

3.2.1 Intf 类

Intf 表示基本的网络接口，比如 h1-eth0 表示 host1 上的 eth0 接口。

属性有：所在节点，名称，连接的link，mac/ip

构造的时候传入节点，端口等属性，并绑定接口到对应的节点端口上。

所支持的方法包括配置 mac/ip 等配置方法，都是通过 ifconfig’ 命令在对应节点上调用 cmd 方法来实现，此外还提供了 config() 方法来一次性配置所有属性

3.2.2 Link 类

Link 类表示一条基本的链路，最基本的链路在 mininet 中就是一堆 veth 接口

创建链路时，需要在两个节点上分别生成两个端口，利用节点和端口，获取对应两个网络接口的名称，例如 s1-sth0 和 h1-eth0，然后调用 makeInePair() 方法，最终调用 util.py 中的 makeIntPair() 方法，调用系统中的 ip link 命令来创造一对 veth pair。

3.2.3 TCIntf 类

被 TC（Linux 下的 traffic control 工具） 自定义的接口，可以配置包括带宽，时延，丢包率，最大队列长度等参数

3.2.4 TCLink

表示一对对称的 TC 接口连接到一起

3.3 Mininet.Node 模块

节点模块表示网络中的基本元素（交换机，主机，控制器），其中，每个主机默认在一个单独的名字空间里，交换机和控制器都在 root 名字空间里

3.3.1 Node 类

Node 是所有网络节点的父类。实现上就是在网络命名空间的一个 shell 进程。可以通过各种管道进行通信。

节点包括名称，是否在网络名字空间，接口，端口等可能的属性

初始函数主要进行参数的初始化，之后通过startShell()启动一个 shell 进程，并从tty 上分离出来，等待传入的命令，句柄被发送给 self.shell 上

addIntf

添加一个接口，如果给定了 port，则建立端口到接口的映射关系。这个映射关系通过 self.inefs 和 self.ports 两个字典分别维护。

cmd

该函数能在节点所在的进程 shell 上执行输入的命令

Config

配置 MAC IP default route 信息

connnectionsTo

返回所有从自身连接到给定节点的接口，即[intf1，intf2]

3.3.2 主机类

包括 Host 和 CPULimitedHost 两个类。

Host 类表示一个主机节点，与 Node类定义相同，在主机类上执行命令通过 Cmd() 或者 sendCmd() 方法，茄汁会等待命令的输出结果，后者直接返回，并允许时候后续的 monitor() 进行监视跟踪

CPULimitedHost 继承自Host 类，通过 cgroup 工具来对 CPU 进行限制

3.3.3 控制器类

Controller 是控制器系欸，表示一个控制器节点。属性有IP 地址，端口等，主要方法包括启动和停止一个控制器

3.3.4 交换机类

Switch 类运行在 Root 运行空间，主要包括 dpid listenPort 属性

3.4 NET 模块

3.4.1 Mininet 类

模拟一个 Mininet 中的网络，包括拓扑，交换机，主机，控制器，链路，接口等，其中最主要的是 build 函数，依次执行：根据拓扑创建网络，配置网络命名空间，配置主机的IP MAC 等信息，检查是否启动 xterm， 是否配置静态 arp 等。

buildFromTopo 方法根据拓扑信息创建节点和相关连接

MininetWithControlNet 类继承自 Mininet’ 类，主要用于用户态 detapath 的时候模拟一个控制器网络，即连接用户态的交换机和用户态的控制器

3.5 Topo 模块

维护网络拓扑的相关信息，除了一些固定结构的拓扑类以外，还提供了 mininet.topolib 模块，提供用户自己创建复杂拓扑相关的库，目前仅包括一个 TreeTopo,给定广度和深度可以自己生成相应标准树拓扑。

MultiGraph 类

表示一个图结构。主要维护节点，边信息，在 MutiGraph 中，节点就是一个序号，边则通过节点和节点对应的连接列表中元素来表示。节点和i二点连接列表的对应关系通过字典来维护。

图结构主要功能是维护一个字典，key 是节点，value 是该节点连接的所有其它节点的连接列表。比如 （h1,[h2,h3,s1]）表示 h1 连接 h2,h3,s1

添加一个节点，实际上就是添加一个key 到 data 字典中。

添加一条边，实际上是添加两个节点，然后将连接信息放到 data 中。需要注意的是一条边的信息仅被保存一次，即放到序号较小的节点对应的 list 中（那找节点较大的边的连接岂不是很麻烦）

Topo 类

拓扑基类，默认的拓扑图被 multigraph 维护，此外还包括节点连接信息，主要的方法是添加节点，添加连接等。Topo 中的一个 node 实际上就是图结构中的一个节点，一个 port 是全局维护增长的源和目的node 对应的序号

addNode 方法是添加节点方法，被 addHost， addSwitch， addConctroller 等方法使用，该方法在图双添加一个节点，然后添加对应节点信息到拓扑参数上

addPort 会同时创建源和目标节点上的端口信息。拓扑中所有的 port 都被self.ports 接口维护，其中 ports[ src ][ dst ] 表示在 src 节点上的 port，该port所在 link连接到 dst 节点，添加一个 port 就是维护了这些信息

addLink 添加一条连接就是添加对应的两个 ports，并在图上添加边

LinearTopo 表示线性拓扑
