C++ 多线程编程

发表于 2019-07-01 | 分类于 C++

线程管理

  1. 启动线程

使用 C++ 线程库启动线程，要构造 std::thread 对象

std::thread 可以调用类型构造，将带有函数调用符类型的实例传入 std::thread 中，替换默认构造函数。
函数对象会复制到新线程中，调用和执行都在新线程的线程空间中进行。

  1. 等待线程的完成

等待线程， join()
分离线程， detach()

可以使用 RAII(资源获取即初始化)，用一个 gurad 对象来管理线程，在析构函数中使用 join()

  1. 向线程传递参数

默认参数要拷贝到线程的独立内存中，即使参数是引用形式
如果期望传一个引用，可以用 std::ref()

除此之外，还可以传递一个成员函数指针作为线程函数，参数是一个对象指针。

  1. 转移线程所有权

C++ 中有很多占有资源可以移动，但不可拷贝，比如 std::ifstream, std::unique_ptr, std::thread
线程的所有权可以在 std::thread 实例中移动，意味着线程的所有权可以在函数外进行转移

std::thread 支持移动的好处是可以创建 thread_guard 类的实例，并拥有线程的所有权

  1. 标识线程

线程的标识是 std::thread::id, 可以通过两种方式进行检索。

get_id()

线程间共享数据

  1. 竞态条件

并发中竞态条件的形成，取决于一个以上的线程的相对执行顺序

  1. 使用互斥量保护共享数据

可以使用 C++ 标准库提供的 std::lock_guard, 使用 RAII 保证了上锁和解锁。

有时候上锁也会暴露出共享数据。

切勿将受到保护的数据指针或引用传递到互斥锁作用域之外

实现一个线程安全的栈

假设栈的每个操作都用互斥锁保护起来，有以下两个问题

  1. empty() 和 size() 是不可靠的

这个两个函数在调用时候是正确的，当它们返回后其它线程就可以自由地访问栈，可能 push，也可能 pop，这样，之前得到的结果就有问题。

  1. top() 和 pop() 之间的问题
Thread A Thread B
if(!s.empty())
if(!s.empty())
int const value = s.top()
int const value = s.top()
s.pop()
do_something(value)
s.pop()
do_something(value)

在运行过程中，运行了两次 top(),让同一个 value 被处理了两次

想要获取 top 元素，然后弹出这个操作，是原子的。

解决办是

  1. 传入一个引用

传入 pop() 想要弹出的值，缺点是要构造一个临时变量

  1. 无异常抛出的拷贝构造函数或移动构造函数

  2. 返回弹出值的指针

指针的优势是自由拷贝，并且不会产生一场。缺点是返回一个指针需要对对象的内存分配进行管理。可以使用 std::shared_Ptr 管理。

C++ 内存模型和原子类型操作

具体的C++ 对象模型再开一个坑

内存模型基本规则

  1. 每一个变量都是一个对象，包括其作为成员变量的对象
  2. 每个对象至少占有一个内存位置
  3. 基本类型都有确定的内存位置
  4. 相邻位域是相同内存中的一部分

C++ 中的原子操作或原子类型

原子操作是一类不可分割的操作，要不完成，要么未完成。

C++标准库提供基本原子类型，对于标准类型进行 typedef T，相关的原子类型就在原来的类型名前加上 atomic_的前缀：atomic_T。

kubernetes 原理

发表于 2019-06-22 | 分类于 kubernetes

Kubernetes 是 Google 基于 Borg 开源的容器编排调度引擎。

Kubernetes 架构

Kubernetes 主要由以下几个核心组件组成

  • etcd: 保存了整个集群的状态
  • apiserver: 提供了资源操作的唯一入口，并提供认证，授权，访问控制，AIPI 注册和发现等机制
  • controller manager: 负责维护集群状态，比如故障检测，自动扩展，滚动更新
  • scheduler: 负责资源的调度，单招预定的调度策略将 Pod 调度到相应的机器上
  • kubelet: 负责维护容器的生命周期，同时也负责 Volume 和 网络的管理
  • Contanner runtime: 负责镜像管理以及 Pod 和容器的真正运行
  • kube-proxy: 为 Service 提供 cluster 内部的服务发现和负载均衡

Cgroups 概念

Cgroups 是 control groups 的缩写，是 Linux 内核提供的一种可以限制，记录，隔离进程组所使用的物理资源的机制。

Cgroups 提供了以下功能:

  1. 限制进程组可以使用的资源数量
  2. 进程组的优先级控制
  3. 记录进程组所使用的资源数量

TCP服务的特点

     TCP相对UDP的特点是:面向连接,字节流和可靠传输

TCP通信双方要建立连接(这种连接是一对一的,基于广播或多播的应用程序不能使用TCP),双方都必须为该连接分配必要的内核资源,以管理连接的状态和连接上的数据.TCP的连接是全双工的.即双方数据读写可以通过一个连接进行.通信双方都必须断开连接以释放系统资源.
