sacnf 和 printf 的缺点 正是 << 和 >> 的强项

写的一个类可以重载 << 和 >> ，然后用 cin 和 cout

3. 使用 new 和 delete 而不是 malloc 和 free

malloc 和 free 的问题在于不知道构造函数和析构函数

比如要构造一个 string 数组

4. 使用 C++ 风格注释

就是使用 // 风格 而不是 /**/ 风格

主要原因是嵌入注释时

5. 对应的 new 和 delete 要采用相同的形式

这样写错在了只 delete掉了第一个对象

new 的时候会发发生两件事情

  1. 内存被分配
  2. 为分配的内存调用一个或多个构造函数

delet 时候也会发生两件事

  1. 为将被释放的内存调用一个或多个析构函数
  2. 释放内存

规则很简单

如果调用了 new 时用了 [], 调用 delete 也要用 []

这个规则有时候可能会被 typedef 掩盖掉

为了避免这个问题，最好杜绝对数组类型用 typedef，使用 vector 来代替数组

6. 析构函数里对指针成员调用 delete

增加一个指针成员要做以下工作

  1. 再每个构造函数里对指针进行初始化，如果 不分配内存，指针要被初始化为空指针。
  2. 删除现有内存，通过赋值操作符分配给指针新内存
  3. 在析构函数里删除指针

另外，删除空指针是安全的，每个指针要么指向空，要么指向有效内存(所以要确保构造函数初始化了指针)，而不用担心是否被new 过，当然，不要用 delete 删除一个没有用 new 来初始化的指针，也不要删除一个传递给你的指针。

除非类成员最初用了 new，否则不用再异构函数 delete

kubernetes 原理

发表于 2019-06-22 | 分类于 kubernetes

Kubernetes 是 Google 基于 Borg 开源的容器编排调度引擎。

Kubernetes 架构

Kubernetes 主要由以下几个核心组件组成

  • etcd: 保存了整个集群的状态
  • apiserver: 提供了资源操作的唯一入口，并提供认证，授权，访问控制，AIPI 注册和发现等机制
  • controller manager: 负责维护集群状态，比如故障检测，自动扩展，滚动更新
  • scheduler: 负责资源的调度，单招预定的调度策略将 Pod 调度到相应的机器上
  • kubelet: 负责维护容器的生命周期，同时也负责 Volume 和 网络的管理
  • Contanner runtime: 负责镜像管理以及 Pod 和容器的真正运行
  • kube-proxy: 为 Service 提供 cluster 内部的服务发现和负载均衡

Cgroups 概念

Cgroups 是 control groups 的缩写，是 Linux 内核提供的一种可以限制，记录，隔离进程组所使用的物理资源的机制。

Cgroups 提供了以下功能:

  1. 限制进程组可以使用的资源数量
  2. 进程组的优先级控制
  3. 记录进程组所使用的资源数量
  4. 进程组的隔离
